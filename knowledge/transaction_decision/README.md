# 二手房交易决策知识包

本目录是供 AI、Agent 和确定性规则执行器复用的机器可读知识源。网页只负责展示，不能替代这里的规则、证据、版本和测试。

## 当前范围

- 城市：北京、深圳、广州、上海、天津、武汉、杭州、苏州、成都、重庆、西安、南京。
- 主体：自然人买卖普通二手住宅。
- 结论：购房行政资格与交易税费；房屋可交易性、贷款审批分别返回边界提示。
- 税种：契税、增值税、增值税附加、个人所得税、个人住房印花税、个人住房土地增值税；上海、重庆持有环节房产税单列。
- 非范围：企业购房、继承、赠与、司法拍卖、非住宅、农村宅基地及无法识别的政策性住房。

## 确定性原则

1. 只有输入完整且命中已审核规则时，才返回 `eligible`、`ineligible` 或精确金额。
2. 缺少主管税务机关确认的计税价格、核定征收率或特殊住房属性时，返回 `unknown` 或 `conditional`。
3. 法定纳税人与合同实际承担人分别记录；买方代付卖方税费不改变法定纳税人。
4. 上海、重庆个人住房房产税属于持有环节，不并入过户时税费总额。
5. 每个结果必须携带规则 ID、政策版本、有效区间和证据 ID。

## 目录

- `manifest.json`：发布清单、城市包和覆盖状态。
- `schemas/`：场景、规则与输出 JSON Schema。
- `national/`：全国共用税收规则及证据。
- `cities/`：12 城资格规则、税收覆盖和本地例外。
- `tests/`：可复算黄金场景。

## 金额口径

全部金额字段以人民币元为单位。精确税费使用税务机关接受的 `taxable_transfer_price`，不擅自用挂牌价、合同价或银行评估价替代。规则执行结果按分四舍五入到 0.01 元，实际申报仍以主管税务机关系统为准。

## 执行与验证

```bash
npm run sync:decision
node scripts/evaluate_transaction_decision.mjs scenario.json
```

`sync:decision` 会依次重建 12 城规则包、生成 348 个黄金场景、执行一致性校验，并导出不含完整测试明细的轻量公网 JSON。当前发布共 70 条可执行规则，348 个场景通过 4,159 项断言。

执行器只接受结构化事实，不从自然语言猜测计税价格、家庭套数或地方核定征收率。LLM 应先做意图识别和字段抽取，再调用规则执行器，并把 `missing_inputs` 转换成追问。
